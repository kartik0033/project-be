## 🚀 Project Overview
**Patient Health System** is a full-stack digital health record management platform. 
- **Core Features:** QR code-based patient identification, OTP-authenticated registration, role-based portals (Patient/Doctor), and medical record management.
- **AI Integration:** Includes an AI Summarizer for medical reports.

---

## 🛠️ Tech Stack
- **Frontend:** React 19 (Vite), React Router v7, Axios, Vanilla CSS (Modern/Premium UI).
- **Backend:** Django 5.2, Django REST Framework, JWT (SimpleJWT).
- **Authentication:** OTP via Twilio (SMS) or Debug mode (console).
- **Database:** SQLite (Development).
- **External Tools:** QR Code generation (`qrcode`), AI Summarization (`sumy`, `pandas`, `scikit-learn`).

## 🛠️ Critical Configurations & Gotchas

### 🌐 CORS & Network
- The Django `CORS_ALLOWED_ORIGINS` must include the specific port Vite is running on (e.g., `http://localhost:5173` or `5174`).
- Current frontend port expectation: **5173** (default).

### 🔑 Environment Variables (`.env`)
- Located in `backend/.env`.
- Contains: `TWILIO` credentials, `SECRET_KEY`, and `DEBUG` status.

### 🛡️ Authentication Flow
1. **Patient:** Register via OTP -> Profile setup -> QR Code generated.
2. **Doctor:** Login via Aadhaar -> Redirected to Doctor Dashboard -> Scan Patient QR.
